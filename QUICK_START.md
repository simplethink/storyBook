# 🚀 快速开始指南

## 第一步：部署云函数（5 分钟）

在微信开发者工具中操作：

### 1. 部署 getUserInfo
```
右键点击 cloudfunctions/getUserInfo
→ 选择"上传并部署：云端安装依赖"
→ 等待部署成功提示
```

### 2. 部署 syncReadingRecords
```
右键点击 cloudfunctions/syncReadingRecords
→ 选择"上传并部署：云端安装依赖"
→ 等待部署成功提示
```

### 3. 部署 syncReviewTasks
```
右键点击 cloudfunctions/syncReviewTasks
→ 选择"上传并部署：云端安装依赖"
→ 等待部署成功提示
```

✅ 验证：在云开发控制台 → 云函数 页面能看到 3 个已部署的函数

---

## 第二步：创建数据库集合（3 分钟）

### 方法一：手动创建（推荐新手）

1. 打开云开发控制台
   - 微信开发者工具顶部工具栏 → 点击"云开发"按钮
   
2. 进入数据库页面
   - 点击左侧"数据库"标签
   
3. 创建集合
   - 点击"+"号
   - 输入集合名称：`users`
   - 权限设置：选择"自定义规则"
   - 粘贴以下规则：
   ```json
   {
     "read": "auth.openid == doc.openid",
     "write": "auth.openid == doc.openid"
   }
   ```
   
4. 重复步骤 3，创建 `reading_records` 和 `review_tasks` 集合

5. 为每个集合添加索引
   - 点击集合 → "索引管理"
   - 添加索引字段：
     - users: openid
     - reading_records: openid, date
     - review_tasks: openid, reviewDate, completed

### 方法二：使用脚本自动创建

修改 `cloudfunctions/quickstartFunctions/index.js`，调用 createCollection 函数。

## 第三步：测试登录功能（2 分钟）

1. 编译运行小程序
   - 点击"编译"按钮
   
2. 进入登录页
   - 应该看到精美的登录界面
   
3. 点击"微信一键登录"
   - 弹出授权对话框
   - 点击"允许"
   
4. 验证用户信息
   - 显示用户头像和昵称
   - 点击"进入首页"
   
5. 检查首页
   - 左上角显示："Hello, 你的昵称"
   - 右侧显示你的头像

✅ 验证成功：能看到用户名和头像

---

## 第四步：测试数据同步（5 分钟）

### 测试读书记录同步

#### 上传测试
1. 添加一条读书记录
   - 首页 → 点击“添加记录”
   - 填写信息（书名、时间等）
   - 点击保存
   
2. 查看云端数据
   - 云开发控制台 → 数据库
   - 点击 `reading_records` 集合
   - 应该能看到刚才添加的记录
   
3. 验证字段
   - 检查 openid 是否正确
   - 检查各字段是否完整

#### 下载测试（新增）✨
1. 清除本地数据（模拟换设备）
   - 微信开发者工具 → 清除缓存
   - 或使用新设备登录
   
2. 打开首页
   - 应该显示“同步数据中...”提示
   - 自动从云端下载所有记录
   
3. 验证数据
   - 查看 localStorage 是否有数据
   - 页面应该显示从云端下载的记录
   - 查看控制台日志确认同步成功

### 测试复习任务生成与同步

1. 查看是否自动生成复习任务
   - 云开发控制台 → 数据库
   - 点击 `review_tasks` 集合
   - 应该能看到 5 条待复习任务（1 天、2 天、4 天、7 天、15 天后）

2. 测试复习任务下载
   - 清除本地缓存
   - 重新打开小程序
   - 复习任务应该自动从云端恢复

✅ 验证成功：云端有对应的记录，多设备数据一致

## 第五步：配置项目（可选）

### 修改环境 ID

如果需要使用其他云环境：

1. 打开 `miniprogram/app.js`
2. 修改 env 参数：
   ```javascript
   env: "你的云环境 ID"
   ```
3. 同时更新所有云函数的 env 配置

### 修改 AppID

如果要发布到其他账号：

1. 打开 `project.config.json`
2. 修改 appid:
   ```json
   "appid": "你的小程序 AppID"
   ```

## 🔍 常见问题排查

### 问题 1：云函数部署失败
**症状:** 上传时提示错误
**解决:**
- 检查网络连接
- 确保已开通云开发服务
- 检查云函数目录结构是否正确
- 重新执行"上传并部署：云端安装依赖"

### 问题 2：获取用户信息失败
**症状:** 点击登录按钮没反应或报错
**解决:**
- 检查基础库版本（需 2.10.4+）
- 确认已在 app.json 添加 login 页面
- 检查 wx.getUserProfile API 调用权限

### 问题 3：数据库操作失败
**症状:** 保存记录时报错
**解决:**
- 检查数据库集合是否已创建
- 验证权限设置是否正确
- 确认 openid 字段是否有值
- 查看云函数日志排查具体错误

### 问题 4：首页不显示用户名
**症状:** 登录后首页还是显示"故事书"
**解决:**
- 检查 app.js 中 getUserInfo 方法是否被调用
- 查看 console 是否有错误日志
- 验证 userInfo 是否正确保存到 globalData
