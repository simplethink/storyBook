# 📊 创建数据库集合指南

## ⚠️ 当前错误

```
DATABASE_COLLECTION_NOT_EXIST: users 集合不存在
```

## 🚀 快速解决（3 分钟）

### 步骤 1：打开云开发控制台

1. 打开 **微信开发者工具**
2. 点击顶部工具栏的 **「云开发」** 按钮
   - 图标：☁️ 云朵形状
3. 进入云开发控制台

### 步骤 2：进入数据库页面

1. 在左侧菜单栏点击 **「数据库」**
2. 看到数据库管理界面

### 步骤 3：创建 users 集合

1. 点击 **"+"** 号（添加集合按钮）
2. 输入集合名称：`users`
3. 点击「确定」

### 步骤 4：设置权限

1. 点击刚创建的 `users` 集合
2. 点击 **「权限设置」** 标签
3. 选择 **「自定义规则」**
4. 粘贴以下 JSON：

```json
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

5. 点击「保存」

### 步骤 5：添加索引（可选但推荐）

1. 点击 **「索引管理」** 标签
2. 点击「添加索引」
3. 字段名：`openid`
4. 类型：`升序` 或 `降序`
5. 勾选 **「唯一索引」**（重要！）
6. 点击「保存」

---

## 📋 需要创建的所有集合

总共需要创建 3 个集合：

### 1. users - 用户信息集合

**权限:**
```json
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

**索引:**
- `openid` (唯一索引)

---

### 2. reading_records - 读书记录集合

**创建步骤:** 重复上面的步骤 3-5，集合名为 `reading_records`

**权限:**
```json
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

**索引:**
- `openid` (升序)
- `date` (升序)
- `openid + date` (复合索引，可选)

---

### 3. review_tasks - 复习任务集合

**创建步骤:** 重复上面的步骤 3-5，集合名为 `review_tasks`

**权限:**
```json
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

**索引:**
- `openid` (升序)
- `reviewDate` (升序)
- `openid + completed` (复合索引，可选)

---

## ✅ 验证创建成功

### 方法 1：在云开发控制台查看

1. 数据库页面应该显示 3 个集合：
   - users
   - reading_records
   - review_tasks

1. 重新编译小程序
2. 点击登录
3. 如果不再报错 `collection not exist`，说明成功

1. 用户登录后，点击 `users` 集合
2. 应该能看到一条用户记录
3. 包含 openid, nickName, avatarUrl 等字段

## 🔧 常见问题

### Q1: 找不到云开发入口？
**A:** 确保已开通云开发服务
- 微信开发者工具 → 云开发
- 如果是第一次，会提示开通
- 按照指引完成开通

### Q2: 创建集合失败？
**A:** 检查以下几点：
- 是否有云开发基础版或更高版本
- 网络是否正常
- 刷新页面重试

### Q3: 权限设置不会填？
**A:** 直接复制上面提供的 JSON 即可
- 所有集合都使用相同的权限规则
- 确保 openid 字段建立唯一索引

### Q4: 索引要不要必须创建？
**A:** 
- 不是必须的，但强烈建议创建
- 没有索引查询会很慢
- 数据量大时会超时
